SQUAD is built for the 90 seconds between deciding to play and stepping on the pitch — find a game, find a venue, find the right squad. The system below is the visual and behavioural contract that holds those moments together across web, iOS, and Android.

01 — Brand & logo

The mark is the brand.

Two strokes, one italic — the SQUAD mark reads as two teams overlapping. Single color always; never gradient, never duotone, never on a busy photo. Wordmark uses Inter Tight 800 italic so anything we ever set on top of it inherits the same forward lean.

Primary lockup · on dark
Primary lockup · on light

Don't

Don't recolour the mark. Brand red on dark, brand red on light, or 100% white silhouette only.

Don't place the lockup on a coloured or photographic background. Always reserve clear space.

Don't add extra slant or rotate. The italic angle is baked into the mark.

02 — Color

One brand. Sport signals. Semantic contracts.

Four layers, stacked: SQUAD red as the single brand color (used at most twice per screen); four functional sport colors for live state, slots, streaks, and losses; a committed slate-charcoal canvas at hue 240 — off-black, never pure black — that makes the warm brand pop and gives the system a temperature of its own; and a semantic alias layer (--color-success, --color-danger, …) that generic UI references so toasts, alerts, and validation never reach into sport tokens directly.

Neutrals · dark— default canvas

Off-black, not pure black. Chroma sits at 0.012–0.020 — above the perceptual threshold (~0.010) so the cool cast is visible, below the level where neutrals start to read as "blue." --bg lifts to 15% L for breathing room; --fg drops to 96% with a faint cool whisper so primary text reads as paper, not screen-glare. Surfaces step up the slate (15 → 19 → 23) instead of leaning on shadows. The result is an athletic dark that has temperature of its own — closer to Whoop / Linear than Vercel-black.

Neutrals · light— alt mode (marketing, venue-CRM)

Same off-white discipline in reverse: --surface is oklch(99.4% 0.004 240) — paper-white, not monitor-pure #fff — so cards feel printed. --bg carries a soft slate wash; --fg is committed slate ink at 18% L instead of neutral black. Semantic aliases — what generic UI references

Toasts, alerts, validation, status badges: these read from semantic aliases, not from sport tokens. A "success" toast points at --color-success; whether it resolves to live-green or something else later, the toast does not need to know. Sport tokens (--live-500, --slot-500) stay for sport contexts — a live game pulse, an open-slots badge, a venue capacity bar.

The atom of the product.

13 — Overlays, motion & elevation

The system has a third dimension.

Cards stay flat — the system is shadow-averse on purpose. Overlays earn the elevation. Four shadow tokens (sm → overlay) live above the surface plane, and three motion tokens (fast / base / slow) decide how they arrive. A global prefers-reduced-motion guard collapses every animation in the system to zero — the live pulse degrades to a static dot.

15 — Iconography

One stroke. Three sizes. Three colors.

Paths are Lucide for general UI and Tabler for sport balls — both 24-grid, ISC/MIT-licensed, and inlined here as raw SVG so the system carries zero runtime dependency. We override the source default of stroke-width 2 to 1.6 for a quieter, more editorial line that pairs with the serif display. Sizes are tokenized as .ic-sm / .ic / .ic-lg (16 / 20 / 24) so a future jump to 18 / 22 / 28 is a single token change. Color is always currentColor — inheriting from the surrounding text — with two opt-in helpers for when an icon needs to disambiguate from its label (.ic--muted, .ic--brand).

Don't
four ways the system breaks
  • Don't mix stroke widths — 1.6 only.
  • Don't use sport tokens (live / win / loss) on chrome icons — those colors carry semantic state.
  • Don't ship multi-color SVGs — duotone reads as ornament; the system stays mono.
  • Don't scale icons with CSS transform — bump the size class so stroke weight stays correct.
16 — In-screen tabs

The other half of navigation.

Bottom-nav (.tabbar / .tab) covers the five global destinations. Everything inside a screen — Upcoming / Past / Hosting, Squad / Substitutes / Waitlist, Stats / Form / Highlights — uses one of two patterns. .pill-tabs is the compact segmented row that sits inside a card or sheet header. .top-tabs is the page-level underline row that scrolls horizontally on mobile. Both carry an optional .count badge.

.pill-tabs
segmented · for compact contexts
.top-tabs
underline · for page-level switches

Overflow scrolls horizontally — scroll snap is off so the underline indicator can travel between tabs without a jump.

When to use which
disambiguating four patterns
2–4 short labels, inside a card .pill-tabs
5+ tabs, or page-level .top-tabs
Filter strip on a list .sport-pill (chip filter, multi-select)
Global destinations .tabbar / .tab (bottom nav)
Count badge contract
when to show a number
  • Mono numerals, tabular-nums, never wraps.
  • Inactive tab → muted-on-surface-2.
  • Active tab → --brand-500 on --brand-soft.
  • Above 99 → render 99+; never four digits.
  • Don't put a count on a tab that doesn't have a number behind it.
17 — Meters & match clock

Progress, in three geometries.

A horizontal .progress bar (already shipped) covers season-fill and slot-filling at row scale. For card and tile scale, a circular .meter reads from across the room — an SVG ring with rounded stroke caps, animated via stroke-dashoffset, drives the fill from a single --p custom property. The .clock variant tints live-green and shows the match minute (72′), with a built-in halftime state for the dead 15.

18 — Badges & notifications

Three pixels that earn their square inch.

A badge is the smallest possible carrier of state. The standalone .badge covers count chips, status dots, and tinted tags anywhere in the UI. For bottom-nav, wrap the icon in .tab-icon-wrap and drop a .badge inside it — that's the only sanctioned way to draw attention to a tab without changing its icon. The badge sits in a 2px-stroke ring of --surface so it cuts cleanly out of a dark or light tabbar.

Rules
five rules to follow
  • One badge per tab, max. Stacking reads as panic.
  • Dot beats count when the message is "something happened" rather than "X things are waiting".
  • Border is always --surface at 2px so the badge stays legible on either surface.
  • Counts over 99 render 99+. Don't go wider than three glyphs.
  • Never tint a non-count badge red just to grab attention — escalation is what .danger is for.
